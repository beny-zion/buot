# רשימת בדיקות לפני פריסה (Deployment Checklist)

## ✅ לפני שמעלים לייצור

### 1. משתני סביבה
- [ ] וודאתם שיש לכם `.env.local` עם ערכי Supabase תקינים
- [ ] וודאתם ש-`.env.local` **לא** מועלה ל-GitHub (צריך להיות ב-`.gitignore`)
- [ ] הכנתם את משתני הסביבה לפלטפורמת ההוסטינג (Vercel/Netlify)

### 2. Supabase
- [ ] יצרתם טבלת `signatures` עם כל השדות
- [ ] הגדרתם Row Level Security (RLS) Policies
- [ ] בדקתם שאתם יכולים להוסיף חתימה דרך האתר
- [ ] בדקתם שהמונה החתימות עובד

### 3. בדיקות תפקוד
- [ ] האתר עולה ב-`npm run dev`
- [ ] טופס ההחתמה עובד
- [ ] מסך התודה מופיע אחרי חתימה
- [ ] כפתור השיתוף לווצאפ עובד
- [ ] הטיימר מוצג ומתעדכן
- [ ] המונה החתימות מתעדכן

### 4. עיצוב ורספונסיביות
- [ ] האתר נראה טוב בנייד (פתחו DevTools + Mobile View)
- [ ] האתר נראה טוב בטאבלט
- [ ] האתר נראה טוב בדסקטופ
- [ ] הפונט העברי נטען תקין
- [ ] הכיוון RTL עובד כמו שצריך

### 5. SEO ושיתוף חברתי
- [ ] התמונות placeholder הוחלפו (אם יש לכם תמונות אמיתיות)
- [ ] המטא-טאגים מוגדרים נכון ב-`layout.js` ו-`page.js`
- [ ] נסיתם לשתף את הלינק בווצאפ - הוא נראה יפה?
- [ ] בדקתם שהכותרת והתיאור נכונים

### 6. ביצועים
- [ ] הריצו `npm run build` ובדקו שאין שגיאות
- [ ] בדקתם את גודל ה-Bundle (צריך להיות קטן יחסית)
- [ ] האתר נטען מהר (פחות מ-2 שניות)

### 7. אבטחה
- [ ] Row Level Security מופעל ב-Supabase
- [ ] אין API Keys חשופים בקוד
- [ ] `.env.local` לא נמצא ב-Git History

---

## 🚀 שלבי הפריסה ב-Vercel

1. **דחפו את הקוד ל-GitHub:**
   ```bash
   git add .
   git commit -m "Ready for deployment"
   git push origin main
   ```

2. **התחברו ל-Vercel:**
   - לכו ל-[vercel.com](https://vercel.com)
   - לחצו "Add New Project"
   - בחרו את הריפו מ-GitHub

3. **הגדירו משתני סביבה:**
   - לחצו על "Environment Variables"
   - הוסיפו:
     - Key: `NEXT_PUBLIC_SUPABASE_URL`
     - Value: (העתיקו מ-`.env.local`)
   - הוסיפו:
     - Key: `NEXT_PUBLIC_SUPABASE_ANON_KEY`
     - Value: (העתיקו מ-`.env.local`)

4. **Deploy:**
   - לחצו "Deploy"
   - המתינו 1-2 דקות
   - האתר שלכם אונליין! 🎉
